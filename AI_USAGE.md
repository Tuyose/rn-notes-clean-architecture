# AI Usage Disclosure

## Statement

AI was used for scaffolding, refactoring suggestions, and documentation drafts. Architecture decisions, debugging, testing, review, and final implementation ownership remain with the developer.

## How AI Was Used

- **Scaffolding:** Initial project structure, boilerplate files, and configuration setup
- **Code generation:** Design system components, domain entities, use cases, and repository implementation
- **Documentation:** README, ARCHITECTURE, ROADMAP, and CHANGELOG drafts
- **Refactoring:** Suggestions for cleaner patterns and TypeScript improvements

## What the Developer Owned

- Architecture decisions (feature-first clean architecture)
- Technology choices (Expo Router, Zustand, React Hook Form + Zod)
- Business logic design (use case boundaries, validation rules)
- Code review and quality assurance
- Test strategy and test case design
- Git commit discipline and history
- Final implementation and debugging

## Why This Matters

This disclosure is included because transparency about AI usage in software development is important. The codebase represents the developer's engineering judgment and architectural decisions, not a purely AI-generated project.
